## 2. README / Descriere proiect

﻿# My Holiday (MVP)

MVP web care coreleaza zboruri, hoteluri si transferuri pentru oferte last-minute in Europa.

## Rulare locala

```bash
npm install
npm run dev
```

Aplicatia este disponibila la `http://localhost:3000/my-holiday`.

## Configurare API-uri

Copiaza `.env.example` in `.env.local` si completeaza cheile. Pentru live data:

```
MY_HOLIDAY_ENABLE_LIVE=true
```

Fara chei, backend-ul foloseste rezultate mock.

## Neon (Postgres) + Prisma

- Creeaza un proiect in Neon si copiaza `DATABASE_URL` in `.env.local`.
- Schema se afla in `prisma/schema.prisma`.
- Cand vrei sa creezi tabelele, ruleaza migrarile Prisma.

## Functionalitati MVP

- Perioada fixa, buget total, plecare din OTP
- Destinatie optionala (mod de explorare)
- Zboruri + hotel + transfer (shuttle/public)
- Costuri defalcate si total
- Lista rezultate cu load more

## Live integration (urmatorul pas)

- Amadeus / Kiwi (zboruri)
- Booking (hoteluri)
- Weather provider (optional)

In `app/api/my-holiday/search/route.ts` se completeaza conectorii reali cand sunt disponibile cheile API.

### 4.3 Calitatea codului

**Aspecte pozitive:**
- Validare input cu Zod schema completă (`SearchSchema`) pe server — bună practică.
- Tipuri TypeScript definite explicit (`SearchPayload`, `HolidayResult`, `LocationCandidate`) — fără `any`.
- Singleton Prisma corect implementat cu `globalThis` pattern pentru dev hot-reload.
- Token caching Amadeus cu verificare expirare (`expiresAt > Date.now() + 60000`) — eficient.
- CSS Modules cu design system coerent (variabile CSS, responsive breakpoint la 720px).
- `useMemo` utilizat corect pentru `payload` în pagina client — evită recalculări inutile.
- Erori Prisma prinse cu `try/catch` fără a bloca răspunsul API.
- `reactCompiler: true` în `next.config.ts` — optimizare modernă activată.

**Probleme de calitate:**
- **Dublă structură de routing** (`app/` + `src/app/`): endpoint-ul `src/app/api/hello/route.js` este inaccesibil și reprezintă cod mort. Trebuie eliminat sau mutat.
- **Hotel și transfer rămân mock și în modul live**: `buildLiveResults()` folosește `randomPick(hotels, index)` și tip transfer hardcodat, creând date înșelătoare pentru utilizator care a activat live mode.
- **Rating calculat artificial**: `3.9 + (index % 6) * 0.1` — nu reflectă rating real.
- **Weather hardcodat**: `randomPick(weatherTags, index)` în ambele moduri — nicio integrare reală.
- **`KIWI_API_KEY` verificat dar neutilizat**: apare în lista `missingKeys` transmisă utilizatorului, dar nu există niciun conector Kiwi în cod — confuzant.
- **Preferences de discovery ignorate în live mode**: `buildLiveResults()` returnează `null` dacă `!payload.destination`, deci modul explore complet nu funcționează cu Amadeus.
- **`MY_HOLIDAY_ENABLE_LIVE` hardcodat în logica API**: condiția `liveEnabled && missingKeys.length === 0` blochează live chiar dacă Kiwi lipsește (deși Kiwi nu e implementat), forțând mereu mock dacă Kiwi key nu e setat.
- **`STRATEGY.md` necompletat**: template cu placeholders `[Define the project vision]`.
- **Mesaj de loading hardcodat**: `'Caut cele mai bune combinatii in Europa...'` — nu reflectă că aplicația suportă și destinații în afara Europei (conform tipologiei de continente din UI: Africa, Asia, America de Sud).

- **Nicio credențială hardcodată în cod** — toate cheile sunt citite din `process.env.*`. Bine.
- **`.env` și `.env.local` excluse din git** (confirmat în STATUS.md). Bine.

- **Nicio autentificare pe endpoint-ul API**: `POST /api/my-holiday/search` este public, fără rate limiting, fără auth token. Oricine poate trimite cereri arbitrare și persista date în Neon.

- **Niciun CSRF protection explicit** — Next.js oferă protecție implicită pentru Server Actions, dar endpoint-urile Route Handler nu au protecție CSRF activă.
- **`console.error` în producție** expune detalii interne despre erorile Amadeus și Prisma în logurile serverului — acceptabil pentru MVP, dar de revizuit la lansare.
- **Token Amadeus stocat în variabilă de modul** (`cachedToken`): în medii serverless (Vercel), variabilele de modul nu persistă între invocații — cache-ul de token nu va funcționa în producție.

### 4.8 Probleme critice identificate

1. **Dublă structură `app/` + `src/app/`** — conflict arhitectural, endpoint din `src/` este inaccesibil.
2. **Token Amadeus incompatibil cu serverless** — `cachedToken` în variabilă de modul nu persiste între invocații Vercel/Lambda.

4. **Endpoint API public fără rate limiting sau autentificare** — expune Neon DB la abuz.
5. **`KIWI_API_KEY` apare ca missing key dar nu există conector** — UX înșelător, utilizatorul e informat că îi lipsește o cheie pentru un serviciu nefuncțional.
